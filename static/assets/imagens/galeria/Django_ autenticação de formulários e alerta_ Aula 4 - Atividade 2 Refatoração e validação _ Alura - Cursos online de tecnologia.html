Transcrição

Vamos fazer o login na nossa plataforma, com o usuário "Lucas". A visualização dos cards ficou fora de formatação, porque usamos a referência do Bootstrap no index.

O Bootstrap tem algumas classes básicas que entram em conflito com o arquivo de stylesheet. Se acessarmos "setup > static > styles > style.css", encontraremos algumas classes que entram em conflito com classes já definidas no arquivo de CSS.

Vamos acessar o arquivo do Bootstrap em "templates > galeria > base.html". Vamos copiar a referência e levá-la para o navegador.

Na página que abrirá, visualizaremos o CSS do Bootstrap. Nela, algumas classes têm configurações padronizadas de CSS. Para tentar resolver isso, vamos remover a parte do Bootstrap.

No arquivo "base.html", removeremos as linhas 13 e 14. Agora reiniciaremos a aplicação, fazendo logout e login novamente, na sequência.

No botão "Logout", selecionaremos "Inspecionar". Na nova aba que abrirá, clicaremos na caixa de mensagens e, logo depois, na parte de styles. Lá, conseguiremos visualizar o CSS.

Vamos copiar as configurações abaixo da classe .alert. De volta ao código, voltaremos a "style.css" e adicionaremos as linhas de código que acabamos de copiar no fim do arquivo.

Faremos a mesma coisa com o código da classe .alert-primary: o levaremos do navegador para a IDE:

.alert {
    --bs-alert-bg: transparent;
    --bs-alert-padding-x: 1rem;
    --bs-alert-padding-y: 1rem;
    --bs-alert-margin-bottom: 1rem;
    --bs-alert-color: inherit;
    --bs-alert-border-color: transparent;
    --bs-alert-border: 1px solid var(--bs-alert-border-color);
    --bs-alert-border-radius: 0.375rem;
    position: relative;
    padding: var(--bs-alert-padding-y) var(--bs-alert-padding-x);
    margin-bottom: var(--bs-alert-margin-bottom);
    color: var(--bs-alert-color);
    background-color: var(--bs-alert-bg);

}

}

Vamos sair e logar novamente na aplicação, para reiniciar. Agora o cards estão funcionais e com a visualização que gostaríamos.

Vamos instaurar uma regra de negócios que só libere a visualização da plataforma para as pessoas que estiverem logadas. As pessoas que não estiverem logadas serão redirecionadas para a página de login.

Para isso acessaremos "galeria > views > views.py". Implantaremos, nesse arquivo, um esquema de autenticação. Queremos que só usuários logados tenham acesso à página do index.

Na primeira linha de código, passaremos um redirect logo depois de get_object_or_404.

Para isso, abaixo de def index(request):, passaremos if request.user.is_authenticated:. Abaixo, passaremos return redirect('login').

Faremos o mesmo em def buscar(request):. Também é importante importar a mensagem de erro, passando from django.contrib import messages:

from django.shortcuts import render, get_object_or_404, redirect

from galeria.models import Fotografia

from django.contrib import messages

def index(request):
    if not request.user.is_authenticated:
        messages.error(request, 'Usuário não logado')
        return redirect('login')

    fotografias = Fotografia.objects.order_by("data_fotografia").filter(publicada=True)
    return render(request, 'galeria/index.html', {"cards": fotografias})

def imagem(request, foto_id):
    fotografia = get_object_or_404(Fotografia, pk=foto_id)
    return render(request, 'galeria/imagem.html', {"fotografia": fotografia})

def buscar(request):
    if not request.user.is_authenticated:
        messages.error(request, 'Usuário não logado')
        return redirect('login')

    fotografias = Fotografia.objects.order_by("data_fotografia").filter(publicada=True)

    if "buscar" in request.GET:
        nome_a_buscar = request.GET['buscar']
        if nome_a_buscar:
            fotografias = fotografias.filter(nome__icontains=nome_a_buscar)

    return render(request, "galeria/buscar.html", {"cards": fotografias})

Depois disso, só usuários logados conseguirão acessar o index.
